# Compelling Minigame Design

「シンプルなのに、つい手が伸びる」ミニゲームを作るための設計書。HTML/JS単体ファイルで完結する小品を想定している。

## 中心となる一文

**掴みは安く、報酬は高く。** 「ついやってしまう」は、安っぽい刺激の連打からは生まれない。**速くて公平なループ**と、**ちゃんと支払われるペイオフ**から生まれる。刺激（juice）は入口であって、中身の代わりではない。これを取り違えた瞬間、ただの安いドーパミン装置になり、バズっても何も残らず、プレイヤーは「あれ、何してたっけ」で去っていく。

だから本スキルのゴールは「中毒」そのものではなく、**プレイヤーが満足して、自分の意思でまた戻ってくる**状態。後述の線引きは付け足しではなく、設計の背骨として扱う。

## 中核ループ（The Core Loop）

良いミニゲームは、ひとつの短いループの反復でできている。理想は **3秒以内** に一周すること：

```
認知（今どうすべきか一目で分かる）
  → 一動作（操作は基本ひとつ）
  → 即時フィードバック（見て・聞いて・触れて分かる）
  → 報酬（点・進捗・気持ちよさ）
  → 少しの変化（次は前回と違う）
  → 認知へ戻る
```

設計の出発点は「華やかさ」ではなく、この一周を**気持ちよく・一瞬で**回せるか。チュートリアル不要、片手・片指で遊べる、起動して2秒で1回目が回る。ここが成立してから装飾を足す。逆順でやると、土台のないゲームに飾りを盛ることになる。

## 「つい」を生む8つのレバー

各レバーは「なぜ効くか」と「どう実装するか」をセットで持つ。全部を最大出力にする必要はない。3〜4個を効かせ、残りは控える。

1. **2秒フック / 即プレイ。** 起動直後にルール3行＋即操作。最初の一手で「お？」を作る。情報のギャップ・不意の動き・見慣れない手触りを最初に置く。スプラッシュやロード待ちで殺さない。

2. **一動作ルール。** 操作は原則ひとつ（タップ、長押し、タイミングよく離す等）。覚えることが少ないほど、思考ではなく反射と判断の遊びになり、回転が上がる。

3. **即時フィードバック（juice）。** 入力の結果を、遅延ゼロで・複数の感覚に返す。画面フラッシュ／パーティクル／数字のポップ／軽い画面シェイク／反応グロー／効果音。「やった」の合図を脳に即返すのがコア。実装は `references/juice-toolkit.md` に雛形。

4. **変動報酬。** 毎回ちょっとだけ違う結果にする。当たり位置のジッター、たまの大当たり（ボーナス／クリティカル）、非線形カーブのクセを毎回変える。「次は何が出る？」が手を止めさせない。ただしこれは諸刃で、ここだけに依存すると搾取側に倒れる（→「線引き」参照）。

5. **遅延報酬／我慢の構造。** これが安いゲームと残るゲームの分水嶺。**すぐ取れる小さな報酬を我慢して、こらえた者に大きな報酬を払う**構造を一本通す。例：早く離すと雀の涙、こらえて適所で離すと高得点。コンボ・連続成功の倍率もこれ（短い弧での伏線回収）。プレイヤーは「上達している」という最良の報酬を得る。

6. **緩急（刺激と余白）。** 全部を最大刺激にすると、人間は逆に何も感じなくなる。各試行の前に短い静かな一拍（"ためる準備"）を置く。連打させない。静寂が次の刺激を効かせる。

7. **エスカレーション。** ラウンドが進むほど、少しずつ速く・狭く・難しく。飽きる前に難度が上がり、習熟が追いかけっこになる。急勾配にしすぎない——理不尽は離脱を生む。

8. **再開したくなる終わり方。** 1プレイは短く。終了画面は、スコア＋"あと一回"を誘う一言＋**意味のある一個の数字**（自己ベスト、こらえた合計時間など、プレイヤー自身の何かを映す指標）。即・再挑戦ボタン。長い反省文は出さない。

## Juice 実装キット（具体の作り方）

`references/juice-toolkit.md` にコピペ可能な雛形がある。最低限おさえる項目：

- **音は Web Audio で合成する。** 外部音源ファイルは使わない（読み込み失敗・著作権・サイズの問題を全部回避できる）。チャージ音（押している間ピッチ上昇）、ヒット音（和音ブリップ）、パーフェクト（上昇アルペジオ）、失敗（ローパスをかけたノイズ＋下降音）。AudioContext は**最初のユーザー操作で初期化／resume** する（自動再生制限）。**ミュート切り替えは必須**。
- **非線形カーブ。** メーター上昇や速度を `値 = (経過/上限)^指数` の累乗にする。指でタイミングを丸暗記させない・終盤に緊張を作る。指数を毎回少しジッターさせると変動報酬にもなる。
- **粒子・リング・シェイク。** 成功で粒子バースト＋衝撃波リング、パーフェクトで二重リング＋星型スパーク、失敗で赤い飛散＋軽いシェイク。シェイクは「やりすぎない」。1フレームごとに減衰させる。
- **色＝情報。** 状態を色で語らせる（安全＝寒色 → 報酬＝金 → 危険＝赤）。装飾ではなく、見ただけで状況が分かる設計。
- **数字を躍らせる。** スコアはポップして弾む、コンボ倍率は跳ねて光る。
- **配慮。** `prefers-reduced-motion` を尊重し、その場合はシェイク・大量パーティクルを自動で抑える。キーボード操作とフォーカスリングも用意する。

## 「中毒性」の健全な線引き

このスキルの題材（ドパガキ的な現象）は、もともと**注意を搾取するコンテンツへの批判**から生まれた言葉だ。だから、その批判が指しているまさにその設計を自分で量産する側に回らないこと。これは道徳の付け足しではなく、**良いゲームと使い捨てのゴミを分ける実用的な基準**でもある。

**基準テスト：遊び終わったあと、プレイヤーの時間は増えたか、減ったか。**
- 何か（達成感・上達・ちょっとした発見）を持って帰れたなら、刺激は正義。
- ただ時間が溶けて「何してたっけ」で終わるなら、作り手が一番安いドーパミンに逃げただけ。

**やらないこと（ダークパターン）：**
- 終われない設計（区切りのない無限ループ、損失回避を煽って引き留める通知地獄）。
- 疑似ギャンブル課金（ガチャ的な確率ガチャ＋"あと一回"課金、ペイ・トゥ・スキップ・ザ・ペイン）。
- 「やめたい」と思っている人を、本人の意思を超えて引き戻す仕掛け。
- とりわけ、**発達途中の未成年**の即時報酬回路を狙い撃ちにする設計。ここは特に慎重に。年齢層が想定できないなら、より安全側に倒す。

**健全な"また遊びたい"とは：** プレイヤーが満足し、自分の意思で戻ってくること。区切りが明確で、いつでも気持ちよく離脱でき、それでも明日また触りたくなる——この状態を目指す。迷ったら「自分の子ども／大切な人にこのループを一日中やらせたいか」を考える。

## 実装の既定値（チェックリスト）

単体HTMLのミニゲームを書くときの初期設定：

- [ ] 単一ファイルで自己完結（外部依存なし、オフラインでも動く）
- [ ] 入力は pointer（マウス＋タッチ）＋キーボードを両対応、`touch-action:none` でスクロール暴発を防ぐ
- [ ] モバイル優先のレイアウト、SVG/Canvas は viewBox / DPR を考慮
- [ ] サンドボックス次第で `localStorage` が使えないことがある。ハイスコア保存はメモリ内を既定にし、永続化は環境を確認してから
- [ ] Web Audio は初回操作で init/resume、ミュート可能
- [ ] `prefers-reduced-motion` を尊重
- [ ] 起動2秒以内に1プレイ目が回るか、最後に必ず確認

## 作例パターン

定番の出発点（いずれも一動作＋遅延報酬を内蔵できる）：

- **チャージ＆リリース：** 押してためて、適所で離す。「早く離す誘惑を我慢」がそのまま遅延報酬の構造になる。非線形カーブと相性が最高。
- **タイミング系：** 動く帯／回る針を狙って止める。精度で報酬を段階化、連続成功でコンボ。
- **ワンボタン回避：** 自動で進む中、ひとつの操作で障害を避ける。距離＝スコア、速度がエスカレーション。
- **マッチ／反射：** 出たものに即応。テンポが上がるほど密度が上がる。

迷ったら「チャージ＆リリース」から始めるとよい。最小の操作で、フック・変動報酬・遅延報酬・緩急・エスカレーションの全レバーを一度に練習できる。

## 作る手順（推奨）

1. ひとつの**中核ループ**を3秒で回せる最小実装を、装飾ゼロで先に作る。手触りが気持ちよくなるまで触る。
2. **遅延報酬の構造**（我慢→大きな報酬）を一本通す。ここが面白さの芯。
3. **即時フィードバック**を足す（音→粒子→フラッシュ→シェイクの順で、過剰になる手前で止める）。
4. **変動報酬とエスカレーション**で飽きを潰す。
5. **緩急**の一拍と、**再開したくなる終了画面**を整える。
6. 最後に**線引きテスト**：これは時間を増やすゲームか、溶かすゲームか。溶かす側に倒れていたら、ペイオフ（中身）側を厚くして直す。
